<div class="package-name">

[@pixi/text-html](../modules/pixi_text_html).HTMLText

</div>


Alternative to [Text](../classes/PIXI.Text.pixi_text) but supports multi-style HTML text. There are few key differences between this and [Text](../classes/PIXI.Text.pixi_text)&#58;  
• HTMLText not support [Internet Explorer](https&#58;//caniuse.com/mdn-svg_elements_foreignobject).  
• Rendering is text asynchronous. If statically rendering, listen to `update` event on BaseTexture.  
• Does not support all style options (e.g., `lineJoin`, `leading`, `textBaseline`, `trim`, `miterLimit`,`fillGradientStops`, `fillGradientType`)


```js
 import { HTMLText } from 'pixi.js';

 const text = new HTMLText("Hello <b>World</b>", { fontSize: 20 });

 text.texture.baseTexture.on('update', () => {
   console.log('Text is redrawn!');
 });
```

### anchor 
**Type:** <code>ObservablePoint</code>

**Description:** The anchor sets the origin point of the sprite. The default value is taken from the [Texture](../classes/PIXI.Texture.pixi_core) and passed to the constructor.

The default is `(0,0)`, this means the sprite's origin is the top left.

Setting the anchor to `(0.5,0.5)` means the sprite's origin is centered.

Setting the anchor to `(1,1)` would mean the sprite's origin point will be the bottom right corner.

If you pass only single parameter, it will set both x and y to the same value as shown in the example below.


```js
 import { Sprite } from 'pixi.js';

 const sprite = new Sprite(Texture.WHITE);
 sprite.anchor.set(0.5); // This will set the origin to center. (0.5) is same as (0.5, 0.5).
```

### interactive 
**Type:** <code>unknown</code>

**Description:** Enable interaction events for the DisplayObject. Touch, pointer and mouse There is 4 types of interaction settings&#58;

* `'none'`&#58; Ignores all interaction events, even on its children.
* `'passive'`&#58; Does not emit events and ignores all hit testing on itself and non-interactive children. Interactive children will still emit events.
* `'auto'`&#58; Does not emit events and but is hit tested if parent is interactive. Same as `interactive = false` in v7
* `'static'`&#58; Emit events and is hit tested. Same as `interaction = true` in v7
* `'dynamic'`&#58; Emits events and is hit tested but will also receive mock interaction events fired from a ticker to allow for interaction when the mouse isn't moving


```js
 import { Sprite } from 'pixi.js';

 const sprite = new Sprite(texture);
 sprite.interactive = 'static';
 sprite.on('tap', (event) => {
     // Handle event
 });
```

### mask 
**Type:** <code>Container &#124; MaskData &#124; null</code>

**Description:** Sets a mask for the displayObject. A mask is an object that limits the visibility of an object to the shape of the mask applied to it. In PixiJS a regular mask must be a [PIXI.Graphics](../classes/PIXI.Graphics.pixi_graphics) or a [PIXI.Sprite](../classes/PIXI.Sprite.pixi_sprite) object. This allows for much faster masking in canvas as it utilities shape clipping. Furthermore, a mask of an object must be in the subtree of its parent. Otherwise, `getLocalBounds` may calculate incorrect bounds, which makes the container's width and height wrong. To remove a mask, set this property to `null`.

For sprite mask both alpha and red channel are used. Black mask is the same as transparent mask.


```js
 import { Graphics, Sprite } from 'pixi.js';

 const graphics = new Graphics();
 graphics.beginFill(0xFF3300);
 graphics.drawRect(50, 250, 100, 100);
 graphics.endFill();

 const sprite = new Sprite(texture);
 sprite.mask = graphics;
```
      

</article>

### getBounds 
**Type:** <code>(skipUpdate&#58; boolean, rect&#58; Rectangle) -&#62; Rectangle</code>

**Description:** Calculates and returns the (world) bounds of the display object as a [Rectangle](../classes/PIXI.Rectangle.pixi_math).
This method is expensive on containers with a large subtree (like the stage). This is because the bounds of a container depend on its children's bounds, which recursively causes all bounds in the subtree to be recalculated. The upside, however, is that calling `getBounds` once on a container will indeed update the bounds of all children (the whole subtree, in fact). This side effect should be exploited by using`displayObject._bounds.getRectangle()` when traversing through all the bounds in a scene graph. Otherwise, calling `getBounds` on each object in a subtree will cause the total cost to increase quadratically as its height increases.

The transforms of all objects in a container's **subtree** and of all **ancestors** are updated. The world bounds of all display objects in a container's **subtree** will also be recalculated.

The `_bounds` object stores the last calculation of the bounds. You can use to entirely skip bounds calculation if needed.

```
const lastCalculatedBounds = displayObject._bounds.getRectangle(optionalRect);

```

Do know that usage of `getLocalBounds` can corrupt the `_bounds` of children (the whole subtree, actually). This is a known issue that has not been solved. See [getLocalBounds](../classes/PIXI.DisplayObject.pixi_display#getLocalBounds) for more details.

`getBounds` should be called with `skipUpdate` equal to `true` in a render() call. This is because the transforms are guaranteed to be update-to-date. In fact, recalculating inside a render() call may cause corruption in certain cases.

#### Parameters:
| Name       | Type                   | Description                                                                                                                                                                          |
| ---------- | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| skipUpdate | <code>boolean</code>   | Setting to `true` will stop the transforms of the scene graph from being updated. This means the calculation returned MAY be out of date BUT will give you a nice performance boost. |
| rect       | <code>Rectangle</code> | Optional rectangle to store the result of the bounds calculation.                                                                                                                    |
#### Returns:
| Type                   | Description                                                                       |
| ---------------------- | --------------------------------------------------------------------------------- |
| <code>Rectangle</code> | * The minimum axis-aligned rectangle in world space that fits around this object. |
</article>

### render 
**Type:** <code>(renderer&#58; Renderer) -&#62; void</code>

**Description:** Renders the object using the WebGL renderer.
The [_render](../classes/PIXI.Container.pixi_display#_render) method is be overriden for rendering the contents of the container itself. This `render` method will invoke it, and also invoke the `render` methods of all children afterward.

If `renderable` or `visible` is false or if `worldAlpha` is not positive or if `cullable` is true and the bounds of this object are out of frame, this implementation will entirely skip rendering. See [PIXI.DisplayObject](../classes/PIXI.DisplayObject.pixi_display) for choosing between `renderable` or `visible`. Generally, setting alpha to zero is not recommended for purely skipping rendering.

When your scene becomes large (especially when it is larger than can be viewed in a single screen), it is advised to employ **culling** to automatically skip rendering objects outside of the current screen. See [cullable](../classes/PIXI.DisplayObject.pixi_display#cullable) and [cullArea](../classes/PIXI.DisplayObject.pixi_display#cullArea). Other culling methods might be better suited for a large number static objects; see [&#64;pixi-essentials/cull](https&#58;//www.npmjs.com/package/&#64;pixi-essentials/cull) and [pixi-cull](https&#58;//www.npmjs.com/package/pixi-cull).

The [renderAdvanced](../classes/PIXI.Container.pixi_display#renderAdvanced) method is internally used when when masking or filtering is applied on a container. This does, however, break batching and can affect performance when masking and filtering is applied extensively throughout the scene graph.

#### Parameters:

| -------- | --------------------- | ------------ |
| renderer | <code>Renderer</code> | The renderer |

### toGlobal 
**Type:** <code>(position&#58; IPointData, point&#58; P, skipUpdate&#58; boolean) -&#62; P</code>

#### Parameters:
| Name       | Type                    | Description                                                                               |

#### Returns:
| Type           | Description                                                |
| -------------- | ---------------------------------------------------------- |
| <code>P</code> | * A point object representing the position of this object. |

### toLocal 
**Type:** <code>(position&#58; IPointData, from&#58; DisplayObject, point&#58; P, skipUpdate&#58; boolean) -&#62; P</code>

**Description:** Calculates the local position of the display object relative to another point.

#### Parameters:
| Name       | Type                       | Description                                                                               |
| ---------- | -------------------------- | ----------------------------------------------------------------------------------------- |
| position   | <code>IPointData</code>    | The world origin to calculate from.                                                       |
| from       | <code>DisplayObject</code> | The DisplayObject to calculate the global position from.                                  |
| point      | <code>P</code>             | A Point object in which to store the value, optional (otherwise will create a new Point). |
| skipUpdate | <code>boolean</code>       | Should we skip the update transform                                                       |
#### Returns:
| Type           | Description                                               |
| -------------- | --------------------------------------------------------- |
| <code>P</code> | * A point object representing the position of this object |

<article>

<article>

### rootContext

## Methods
<article>

### clear 
**Type:** <code>(clearColor&#58; string, alpha&#58; number) -&#62; void</code>

**Description:** Clear the canvas of renderer.

#### Parameters:
| Name       | Type                | Description                                                         |
| ---------- | ------------------- | ------------------------------------------------------------------- |
| clearColor | <code>string</code> | Clear the canvas with this color, except the canvas is transparent. |
| alpha      | <code>number</code> | Alpha to apply to the background fill color.                        |
</article>


---------------------------------------

<article>

### init

---------------------------------------

<article>

### invalidateBlendMode 
**Type:** <code>() -&#62; void</code>

**Description:** Checks if blend mode has changed.

</article>


---------------------------------------

<article>

### setBlendMode 
**Type:** <code>(blendMode&#58; number, readyForOuterBlend&#58; boolean) -&#62; void</code>

**Description:** Sets the blend mode of the renderer.

#### Parameters:
| Name               | Type                 | Description                                                                                                      |
| ------------------ | -------------------- | ---------------------------------------------------------------------------------------------------------------- |
| blendMode          | <code>number</code>  | See [PIXI.BLEND_MODES](../enums/PIXI.BLEND_MODES.pixi_constants) for valid values.                               |
| readyForOuterBlend | <code>boolean</code> | Some blendModes are dangerous, they affect outer space of sprite. Pass `true` only if you are ready to use them. |
</article>


---------------------------------------

<article>

### setContextTransform 
**Type:** <code>(transform&#58; Matrix, roundPixels&#58; boolean, localResolution&#58; number) -&#62; void</code>

**Description:** Sets matrix of context. called only from render() methods takes care about resolution

#### Parameters:
| Name            | Type                 | Description                                                           |
| --------------- | -------------------- | --------------------------------------------------------------------- |
| transform       | <code>Matrix</code>  | world matrix of current element                                       |
| roundPixels     | <code>boolean</code> | whether to round (tx,ty) coords                                       |
| localResolution | <code>number</code>  | If specified, used instead of `renderer.resolution` for local scaling |
</article>

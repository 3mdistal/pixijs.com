[@pixi/canvas-renderer](../modules/pixi_canvas_renderer).CanvasRenderer

The CanvasRenderer draws the scene and all its content onto a 2d canvas.
This renderer should be used for browsers that support WebGL.

This renderer should be used for browsers that do not support WebGL. Don't forget to add the view to your DOM or you will not see anything!

Renderer is composed of systems that manage specific tasks. The following systems are added by default whenever you create a renderer&#58;

&#124; System &#124; Description &#124;
&#124; ------ &#124; ----------- &#124;

&#124; Generic Systems                                                               &#124; Systems that manage functionality that all renderer types share        &#124;
&#124; ----------------------------------------------------------------------------- &#124; ---------------------------------------------------------------------- &#124;
&#124; [PIXI.ViewSystem](../classes/PIXI.ViewSystem.pixi_core)                       &#124; This manages the main view of the renderer usually a Canvas            &#124;
&#124; [PIXI.PluginSystem](../classes/PIXI.PluginSystem.pixi_core)                   &#124; This manages plugins for the renderer                                  &#124;
&#124; [PIXI.BackgroundSystem](../classes/PIXI.BackgroundSystem.pixi_core)           &#124; This manages the main views background color and alpha                 &#124;
&#124; [PIXI.StartupSystem](../classes/PIXI.StartupSystem.pixi_core)                 &#124; Boots up a renderer and initiatives all the systems                    &#124;
&#124; [PIXI.EventSystem](../classes/PIXI.EventSystem.pixi_events)                   &#124; This manages UI events.                                                &#124;
&#124; [PIXI.GenerateTextureSystem](../classes/PIXI.GenerateTextureSystem.pixi_core) &#124; This adds the ability to generate textures from any PIXI.DisplayObject &#124;

&#124; PixiJS High-Level Systems                                                            &#124; Set of specific systems designed to work with PixiJS objects &#124;
&#124; ------------------------------------------------------------------------------------ &#124; ------------------------------------------------------------ &#124;
&#124; [PIXI.CanvasContextSystem](../classes/PIXI.CanvasContextSystem.pixi_canvas_renderer) &#124; This manages the canvas 2d contexts and their state          &#124;
&#124; [PIXI.CanvasMaskSystem](../classes/PIXI.CanvasMaskSystem.pixi_canvas_renderer)       &#124; This manages masking operations.                             &#124;
&#124; [PIXI.CanvasExtract](../classes/PIXI.CanvasExtract.pixi_canvas_extract)              &#124; This extracts image data from a PIXI.DisplayObject           &#124;
&#124; [PIXI.CanvasPrepare](../classes/PIXI.CanvasPrepare.pixi_canvas_prepare)              &#124; This prepares a PIXI.DisplayObject async for rendering       &#124;

The breadth of the API surface provided by the renderer is contained within these systems.

### clearBeforeRender: <span><code>boolean</code></span>
This sets if the CanvasRenderer will clear the canvas or not before the new render pass. If the scene is NOT transparent PixiJS will use a canvas sized fillRect operation every frame to set the canvas background color. If the scene is transparent PixiJS will use clearRect to clear the canvas every frame. Disable this by setting this to false. For example, if your game has a canvas filling background image you often don't need this set.

### emitWithCustomOptions: <span><code>(runner&#58; Runner, options&#58; Record&#60;string, unknown&#62;) -&#62; void</code></span>
A function that will run a runner and call the runners function but pass in different options to each system based on there name.

E.g. If you have two systems added called `systemA` and `systemB` you could call do the following&#58;

```
system.emitWithCustomOptions(init, {
    systemA: {...optionsForA},
    systemB: {...optionsForB},
});

```

`init` would be called on system A passing `optionsForA` and on system B passing `optionsForB`.

#### Parameters:
| Name    | Type                                         | Description                       |
| ------- | -------------------------------------------- | --------------------------------- |
| runner  | <code>Runner</code>                          | the runner to target              |
| options | <code>Record&#60;string, unknown&#62;</code> | key value options for each system |
</article>

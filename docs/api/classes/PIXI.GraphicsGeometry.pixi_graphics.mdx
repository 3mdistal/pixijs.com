[@pixi/graphics](../modules/pixi_graphics).GraphicsGeometry


The Graphics class contains methods used to draw primitive shapes such as lines, circles and rectangles to the display, and to color and fill them.

GraphicsGeometry is designed to not be continually updating the geometry since it's expensive to re-tesselate using **earcut**. Consider using [PIXI.Mesh](../classes/PIXI.Mesh.pixi_mesh) for this use-case, it's much faster.

### bounds: <span><code>Bounds</code></span>
Get the current bounds of the graphic geometry.
Since 6.5.0, bounds of the graphics geometry are calculated based on the vertices of generated geometry. Since shapes or strokes with full transparency (`alpha: 0`) will not generate geometry, they are not considered when calculating bounds for the graphics geometry. See PR [#8343](https&#58;//github.com/pixijs/pixijs/pull/8343) and issue [#8623](https&#58;//github.com/pixijs/pixijs/pull/8623).

### addAttribute: <span><code>(id&#58; string, buffer&#58; PIXI.Buffer &#124; Array&#60;number&#62;, size&#58; number, normalized&#58; boolean, type&#58; TYPES, stride&#58; number, start&#58; number, instance&#58; boolean) -&#62; this</code></span>
Adds an attribute to the geometry Note&#58; `stride` and `start` should be `undefined` if you dont know them, not 0!

#### Parameters:
| Name       | Type                                                  | Description                                                                                                            |
| ---------- | ----------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| id         | <code>string</code>                                   | the name of the attribute (matching up to a shader)                                                                    |
| buffer     | <code>PIXI.Buffer &#124; Array&#60;number&#62;</code> | the buffer that holds the data of the attribute . You can also provide an Array and a buffer will be created from it.  |
| size       | <code>number</code>                                   | the size of the attribute. If you have 2 floats per vertex (eg position x and y) this would be 2                       |
| normalized | <code>boolean</code>                                  | should the data be normalized.                                                                                         |
| type       | <code>TYPES</code>                                    | what type of number is the attribute. Check [PIXI.TYPES](../enums/PIXI.TYPES.pixi_constants) to see the ones available |
| stride     | <code>number</code>                                   | How far apart, in bytes, the start of each value is. (used for interleaving data)                                      |
| start      | <code>number</code>                                   | How far into the array to start reading values (used for interleaving data)                                            |
| instance   | <code>boolean</code>                                  | Instancing flag                                                                                                        |
#### Returns:
| Type              | Description                          |
| ----------------- | ------------------------------------ |
| <code>this</code> | * Returns self, useful for chaining. |

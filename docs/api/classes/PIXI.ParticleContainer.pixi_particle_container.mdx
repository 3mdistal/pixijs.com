<div class="package-name">

[@pixi/particle-container](../modules/pixi_particle_container).ParticleContainer

</div>


The ParticleContainer class is a really fast version of the Container built solely for speed, so use when you need a lot of sprites or particles.

The tradeoff of the ParticleContainer is that most advanced functionality will not work. ParticleContainer implements the basic object transform (position, scale, rotation) and some advanced functionality like tint (as of v4.5.6).

Other more advanced functionality like masking, children, filters, etc will not work on sprites in this batch.

It's extremely easy to use. And here you have a hundred sprites that will be rendered at the speed of light.


```js
 import { ParticleContainer, Sprite } from 'pixi.js';

 const container = new ParticleContainer();

 for (let i = 0; i < 100; ++i)
 {
     let sprite = Sprite.from('myImage.png');
     container.addChild(sprite);
 }
```
      

        

<article>

### Constructor:
<h4><code>new PIXI.ParticleContainer(maxSize&#58; number, properties&#58; IParticleProperties, batchSize&#58; number, autoResize&#58; boolean) </code></h4>

#### Parameters:
| Name                | Type                             | Default            | Description                                                                                                             |
| ------------------- | -------------------------------- | ------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| maxSize             | <code>number</code>              | <code>1500</code>  | The maximum number of particles that can be rendered by the container. Affects size of allocated buffers.               |
| properties          | <code>IParticleProperties</code> | <code></code>      | The properties of children that should be uploaded to the gpu and applied.                                              |
| properties.vertices | <code>boolean</code>             | <code>false</code> | When true, vertices be uploaded and applied. if sprite's ` scale/anchor/trim/frame/orig` is dynamic, please set `true`. |
| properties.position | <code>boolean</code>             | <code>true</code>  | When true, position be uploaded and applied.                                                                            |
| properties.rotation | <code>boolean</code>             | <code>false</code> | When true, rotation be uploaded and applied.                                                                            |
| properties.uvs      | <code>boolean</code>             | <code>false</code> | When true, uvs be uploaded and applied.                                                                                 |
| properties.tint     | <code>boolean</code>             | <code>false</code> | When true, alpha and tint be uploaded and applied.                                                                      |
| batchSize           | <code>number</code>              | <code>16384</code> | Number of particles per batch. If less than maxSize, it uses maxSize instead.                                           |
| autoResize          | <code>boolean</code>             | <code>false</code> | If true, container allocates more batches in case there are more than `maxSize` particles.                              |

## Members
<article>

### autoResize 
**Type:** <code>boolean</code>

**Description:** If true, container allocates more batches in case there are more than `maxSize` particles.

<article>

### blendMode 
**Type:** <code>BLEND_MODES</code>

**Default Value**: <code>PIXI.BLEND_MODES.NORMAL</code>

**Description:** The blend mode to be applied to the sprite. Apply a value of `PIXI.BLEND_MODES.NORMAL`to reset the blend mode.

</article>


---------------------------------------

<article>

### roundPixels 
**Type:** <code>boolean</code>

**Default Value**: <code>true</code>

**Description:** If true PixiJS will Math.floor() x/y values when rendering, stopping pixel interpolation. Advantages can include sharper image quality (like text) and faster rendering on canvas. The main disadvantage is movement of objects may appear less smooth. Default to true here as performance is usually the priority for particles.

</article>


---------------------------------------

<article>

### tint 
**Type:** <code>ColorSource</code>

**Default Value**: <code>0xFFFFFF</code>

**Description:** The tint applied to the container. This is a hex value. A value of 0xFFFFFF will remove any tint effect. IMPORTANT&#58; This is a WebGL only feature and will be ignored by the canvas renderer.

</article>

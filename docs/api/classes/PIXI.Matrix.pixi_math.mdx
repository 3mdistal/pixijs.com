<div class="package-name">

[@pixi/math](../modules/pixi_math).Matrix

</div>

The PixiJS Matrix as a class makes it a lot faster.
Here is a representation of it&#58;

```
| a | c | tx|
| b | d | ty|
| 0 | 0 | 1 |

```



        

<article>

### Constructor:
<h4><code>new PIXI.Matrix(a&#58; number, b&#58; number, c&#58; number, d&#58; number, tx&#58; number, ty&#58; number) </code></h4>

#### Parameters:
| Name | Type                | Default        | Description   |
| ---- | ------------------- | -------------- | ------------- |
| a    | <code>number</code> | <code>1</code> | x scale       |
| b    | <code>number</code> | <code>0</code> | y skew        |
| c    | <code>number</code> | <code>0</code> | x skew        |
| d    | <code>number</code> | <code>1</code> | y scale       |
| tx   | <code>number</code> | <code>0</code> | x translation |
| ty   | <code>number</code> | <code>0</code> | y translation |

</article>

### apply 
**Type:** <code>(pos&#58; IPointData, newPos&#58; PIXI.Point) -&#62; PIXI.Point</code>

**Description:** Get a new position with the current transformation applied. Can be used to go from a child's coordinate space to the world coordinate space. (e.g. rendering)

#### Parameters:
| Name   | Type                    | Description                                                                  |
| ------ | ----------------------- | ---------------------------------------------------------------------------- |
| pos    | <code>IPointData</code> | The origin                                                                   |
| newPos | <code>PIXI.Point</code> | The point that the new position is assigned to (allowed to be same as input) |
#### Returns:
| Type                    | Description                                    |
| ----------------------- | ---------------------------------------------- |
| <code>PIXI.Point</code> | The new point, transformed through this matrix |
</article>

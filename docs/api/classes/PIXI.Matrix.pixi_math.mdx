[@pixi/math](../modules/pixi_math).Matrix

The PixiJS Matrix as a class makes it a lot faster.
Here is a representation of it&#58;

```
| a | c | tx|
| b | d | ty|
| 0 | 0 | 1 |

```

#### Parameters:
| Name | Type                | Default        | Description   |
| ---- | ------------------- | -------------- | ------------- |
| a    | <code>number</code> | <code>1</code> | x scale       |

| tx   | <code>number</code> | <code>0</code> | x translation |
| ty   | <code>number</code> | <code>0</code> | y translation |

### apply: <span><code>(pos&#58; IPointData, newPos&#58; PIXI.Point) -&#62; PIXI.Point</code></span>
Get a new position with the current transformation applied. Can be used to go from a child's coordinate space to the world coordinate space. (e.g. rendering)

#### Parameters:
| Name   | Type                    | Description                                                                  |
| ------ | ----------------------- | ---------------------------------------------------------------------------- |
| pos    | <code>IPointData</code> | The origin                                                                   |
| newPos | <code>PIXI.Point</code> | The point that the new position is assigned to (allowed to be same as input) |
#### Returns:
| Type                    | Description                                    |
| ----------------------- | ---------------------------------------------- |
| <code>PIXI.Point</code> | The new point, transformed through this matrix |
</article>

### applyInverse: <span><code>(pos&#58; IPointData, newPos&#58; PIXI.Point) -&#62; PIXI.Point</code></span>

#### Parameters:
| Name   | Type                    | Description                                                                  |
| ------ | ----------------------- | ---------------------------------------------------------------------------- |
| pos    | <code>IPointData</code> | The origin                                                                   |
| newPos | <code>PIXI.Point</code> | The point that the new position is assigned to (allowed to be same as input) |
#### Returns:
| Type                    | Description                                            |
| ----------------------- | ------------------------------------------------------ |
| <code>PIXI.Point</code> | The new point, inverse-transformed through this matrix |
</article>

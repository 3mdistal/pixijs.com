The Geometry represents a model. It consists of two components&#58;

* GeometryStyle - The structure of the model such as the attributes layout
* GeometryData - the data of the model - this consists of buffers. This can include anything from positions, uvs, normals, colors etc.

Geometry can be defined without passing in a style or data if required (thats how I prefer!)


```js
 import { Geometry } from 'pixi.js';

 const geometry = new Geometry();

 geometry.addAttribute('positions', [0, 0, 100, 0, 100, 100, 0, 100], 2);
 geometry.addAttribute('uvs', [0, 0, 1, 0, 1, 1, 0, 1], 2);
 geometry.addIndex([0, 1, 2, 1, 3, 2]);
```

### addAttribute 
**Type:** <code>(id&#58; string, buffer&#58; PIXI.Buffer &#124; Array&#60;number&#62;, size&#58; number, normalized&#58; boolean, type&#58; TYPES, stride&#58; number, start&#58; number, instance&#58; boolean) -&#62; this</code>

**Description:** Adds an attribute to the geometry Note&#58; `stride` and `start` should be `undefined` if you dont know them, not 0!

#### Parameters:
| Name       | Type                                                  | Description                                                                                                            |
| ---------- | ----------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| id         | <code>string</code>                                   | the name of the attribute (matching up to a shader)                                                                    |
| buffer     | <code>PIXI.Buffer &#124; Array&#60;number&#62;</code> | the buffer that holds the data of the attribute . You can also provide an Array and a buffer will be created from it.  |
| size       | <code>number</code>                                   | the size of the attribute. If you have 2 floats per vertex (eg position x and y) this would be 2                       |
| normalized | <code>boolean</code>                                  | should the data be normalized.                                                                                         |
| type       | <code>TYPES</code>                                    | what type of number is the attribute. Check [PIXI.TYPES](../enums/PIXI.TYPES.pixi_constants) to see the ones available |
| stride     | <code>number</code>                                   | How far apart, in bytes, the start of each value is. (used for interleaving data)                                      |
| start      | <code>number</code>                                   | How far into the array to start reading values (used for interleaving data)                                            |
| instance   | <code>boolean</code>                                  | Instancing flag                                                                                                        |
#### Returns:
| Type              | Description                          |
| ----------------- | ------------------------------------ |
| <code>this</code> | * Returns self, useful for chaining. |

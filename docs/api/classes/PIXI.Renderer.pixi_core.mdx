[@pixi/core](../modules/pixi_core).Renderer

The Renderer draws the scene and all its content onto a WebGL enabled canvas.
This renderer should be used for browsers that support WebGL.

This renderer works by automatically managing WebGLBatches, so no need for Sprite Batches or Sprite Clouds. Don't forget to add the view to your DOM or you will not see anything!

Renderer is composed of systems that manage specific tasks. The following systems are added by default whenever you create a renderer&#58;

&#124; System &#124; Description &#124;
&#124; ------ &#124; ----------- &#124;

&#124; Generic Systems                                                     &#124; Systems that manage functionality that all renderer types share &#124;
&#124; ------------------------------------------------------------------- &#124; --------------------------------------------------------------- &#124;
&#124; [PIXI.ViewSystem](../classes/PIXI.ViewSystem.pixi_core)             &#124; This manages the main view of the renderer usually a Canvas     &#124;
&#124; [PIXI.PluginSystem](../classes/PIXI.PluginSystem.pixi_core)         &#124; This manages plugins for the renderer                           &#124;
&#124; [PIXI.BackgroundSystem](../classes/PIXI.BackgroundSystem.pixi_core) &#124; This manages the main views background color and alpha          &#124;
&#124; [PIXI.StartupSystem](../classes/PIXI.StartupSystem.pixi_core)       &#124; Boots up a renderer and initiatives all the systems             &#124;
&#124; [PIXI.EventSystem](../classes/PIXI.EventSystem.pixi_events)         &#124; This manages UI events.                                         &#124;

&#124; WebGL Core Systems                                                    &#124; Provide an optimised, easy to use API to work with WebGL                    &#124;
&#124; --------------------------------------------------------------------- &#124; --------------------------------------------------------------------------- &#124;
&#124; [PIXI.ContextSystem](../classes/PIXI.ContextSystem.pixi_core)         &#124; This manages the WebGL context and extensions.                              &#124;
&#124; [PIXI.FramebufferSystem](../classes/PIXI.FramebufferSystem.pixi_core) &#124; This manages framebuffers, which are used for offscreen rendering.          &#124;
&#124; [PIXI.GeometrySystem](../classes/PIXI.GeometrySystem.pixi_core)       &#124; This manages geometries & buffers, which are used to draw object meshes.    &#124;
&#124; [PIXI.ShaderSystem](../classes/PIXI.ShaderSystem.pixi_core)           &#124; This manages shaders, programs that run on the GPU to calculate 'em pixels. &#124;
&#124; [PIXI.StateSystem](../classes/PIXI.StateSystem.pixi_core)             &#124; This manages the WebGL state variables like blend mode, depth testing, etc. &#124;
&#124; [PIXI.TextureSystem](../classes/PIXI.TextureSystem.pixi_core)         &#124; This manages textures and their resources on the GPU.                       &#124;
&#124; [PIXI.TextureGCSystem](../classes/PIXI.TextureGCSystem.pixi_core)     &#124; This will automatically remove textures from the GPU if they are not used.  &#124;
&#124; [PIXI.MultisampleSystem](../classes/PIXI.MultisampleSystem.pixi_core) &#124; This manages the multisample const on the WEbGL Renderer                    &#124;

&#124; PixiJS High-Level Systems                                                     &#124; Set of specific systems designed to work with PixiJS objects                                                    &#124;
&#124; ----------------------------------------------------------------------------- &#124; --------------------------------------------------------------------------------------------------------------- &#124;
&#124; [PIXI.GenerateTextureSystem](../classes/PIXI.GenerateTextureSystem.pixi_core) &#124; This adds the ability to generate textures from any PIXI.DisplayObject                                          &#124;
&#124; [PIXI.ProjectionSystem](../classes/PIXI.ProjectionSystem.pixi_core)           &#124; This manages the projectionMatrix, used by shaders to get NDC coordinates.                                      &#124;
&#124; [PIXI.RenderTextureSystem](../classes/PIXI.RenderTextureSystem.pixi_core)     &#124; This manages render-textures, which are an abstraction over framebuffers.                                       &#124;
&#124; [PIXI.MaskSystem](../classes/PIXI.MaskSystem.pixi_core)                       &#124; This manages masking operations.                                                                                &#124;
&#124; [PIXI.ScissorSystem](../classes/PIXI.ScissorSystem.pixi_core)                 &#124; This handles scissor masking, and is used internally by [PIXI.MaskSystem](../classes/PIXI.MaskSystem.pixi_core) &#124;
&#124; [PIXI.StencilSystem](../classes/PIXI.StencilSystem.pixi_core)                 &#124; This handles stencil masking, and is used internally by [PIXI.MaskSystem](../classes/PIXI.MaskSystem.pixi_core) &#124;
&#124; [PIXI.FilterSystem](../classes/PIXI.FilterSystem.pixi_core)                   &#124; This manages the filtering pipeline for post-processing effects.                                                &#124;
&#124; [PIXI.BatchSystem](../classes/PIXI.BatchSystem.pixi_core)                     &#124; This manages object renderers that defer rendering until a flush.                                               &#124;
&#124; [PIXI.Prepare](../classes/PIXI.Prepare.pixi_prepare)                          &#124; This manages uploading assets to the GPU.                                                                       &#124;
&#124; [PIXI.Extract](../classes/PIXI.Extract.pixi_extract)                          &#124; This extracts image data from display objects.                                                                  &#124;

The breadth of the API surface provided by the renderer is contained within these systems.

<article>

### emitWithCustomOptions 
**Type:** <code>(runner&#58; Runner, options&#58; Record&#60;string, unknown&#62;) -&#62; void</code>

**Description:** A function that will run a runner and call the runners function but pass in different options to each system based on there name.

E.g. If you have two systems added called `systemA` and `systemB` you could call do the following&#58;

```
system.emitWithCustomOptions(init, {
    systemA: {...optionsForA},
    systemB: {...optionsForB},
});

```

`init` would be called on system A passing `optionsForA` and on system B passing `optionsForB`.

#### Parameters:
| Name    | Type                                         | Description                       |
| ------- | -------------------------------------------- | --------------------------------- |
| runner  | <code>Runner</code>                          | the runner to target              |
| options | <code>Record&#60;string, unknown&#62;</code> | key value options for each system |
</article>

### generateTexture 
**Type:** <code>(displayObject&#58; IRenderableObject, options&#58; object) -&#62; RenderTexture</code>

**Description:** Useful function that returns a texture of the display object that can then be used to create sprites This can be quite useful if your displayObject is complicated and needs to be reused multiple times.

#### Parameters:
| Name                | Type                           | Description                                                                                                                            |
| ------------------- | ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------- |
| displayObject       | <code>IRenderableObject</code> | The displayObject the object will be generated from.                                                                                   |
| options             | <code>object</code>            | Generate texture options.                                                                                                              |
| options.scaleMode   | <code>PIXI.SCALE_MODES</code>  | The scale mode of the texture.                                                                                                         |
| options.resolution  | <code>number</code>            | The resolution / device pixel ratio of the texture being generated.                                                                    |
| options.region      | <code>PIXI.Rectangle</code>    | The region of the displayObject, that shall be rendered, if no region is specified, defaults to the local bounds of the displayObject. |
| options.multisample | <code>PIXI.MSAA_QUALITY</code> | The number of samples of the frame buffer.                                                                                             |
#### Returns:
| Type                       | Description                       |
| -------------------------- | --------------------------------- |
| <code>RenderTexture</code> | A texture of the graphics object. |
</article>

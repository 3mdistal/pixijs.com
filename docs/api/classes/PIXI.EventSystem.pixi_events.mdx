<article>

### autoPreventDefault 
**Type:** <code>boolean</code>

**Default Value**: <code>true</code>

**Description:** Should default browser actions automatically be prevented. Does not apply to pointer events for backwards compatibility preventDefault on pointer events stops mouse events from firing Thus, for every pointer event, there will always be either a mouse of touch event alongside it.

</article>


---------------------------------------

<article>

### cursorStyles 
**Type:** <code>Record&#60;string, string &#124; ((mode&#58; string) =&#62; void) &#124; CSSStyleDeclaration&#62;</code>

**Description:** Dictionary of how different cursor modes are handled. Strings are handled as CSS cursor values, objects are handled as dictionaries of CSS values for `domElement`, and functions are called instead of changing the CSS. Default CSS cursor values are provided for 'default' and 'pointer' modes.

</article>


---------------------------------------

### domElement 
**Type:** <code>HTMLElement</code>

**Default Value**: <code>undefined</code>

**Description:** The DOM element to which the root event listeners are bound. This is automatically set to the renderer's [view](../classes/PIXI.Renderer.pixi_core#view).

</article>

<article>

### resolution 
**Type:** <code>number</code>

**Default Value**: <code>1</code>

**Description:** The resolution used to convert between the DOM client space into world space.

</article>


---------------------------------------

<article>

### rootBoundary 
**Type:** <code>EventBoundary</code>

**Description:** The [PIXI.EventBoundary](../classes/PIXI.EventBoundary.pixi_events) for the stage.
The [rootTarget](../classes/PIXI.EventBoundary.pixi_events#rootTarget) of this root boundary is automatically set to the last rendered object before any event processing is initiated. This means the main scene needs to be rendered atleast once before UI events will start propagating.

The root boundary should only be changed during initialization. Otherwise, any state held by the event boundary may be lost (like hovered & pressed DisplayObjects).

</article>

<article>

### mapPositionToPoint 
**Type:** <code>(point&#58; PIXI.IPointData, x&#58; number, y&#58; number) -&#62; void</code>

**Description:** Maps x and y coords from a DOM object and maps them correctly to the PixiJS view. The resulting value is stored in the point. This takes into account the fact that the DOM element could be scaled and positioned anywhere on the screen.

#### Parameters:
| Name  | Type                         | Description                                 |
| ----- | ---------------------------- | ------------------------------------------- |
| point | <code>PIXI.IPointData</code> | the point that the result will be stored in |
| x     | <code>number</code>          | the x coord of the position to map          |
| y     | <code>number</code>          | the y coord of the position to map          |
</article>

<article>

### normalizeWheelEvent 
**Type:** <code>(nativeEvent&#58; WheelEvent) -&#62; FederatedWheelEvent</code>

**Description:** Normalizes the native [WheelEvent](https&#58;//w3c.github.io/uievents/#interface-wheelevent).
The returned [PIXI.FederatedWheelEvent](../classes/PIXI.FederatedWheelEvent.pixi_events) is a shared instance. It will not persist across multiple native wheel events.

#### Parameters:
| Name        | Type                    | Description                                         |
| ----------- | ----------------------- | --------------------------------------------------- |
| nativeEvent | <code>WheelEvent</code> | The native wheel event that occurred on the canvas. |
#### Returns:
| Type                             | Description              |
| -------------------------------- | ------------------------ |
| <code>FederatedWheelEvent</code> | A federated wheel event. |
</article>

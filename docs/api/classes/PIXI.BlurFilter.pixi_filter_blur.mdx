<div class="package-name">

[@pixi/filter-blur](../modules/pixi_filter_blur).BlurFilter

</div>

The BlurFilter applies a Gaussian blur to an object.
The strength of the blur can be set for the x-axis and y-axis separately.



        

<article>

### Constructor:
<h4><code>new PIXI.BlurFilter(strength&#58; number, quality&#58; number, resolution&#58; , kernelSize&#58; number) </code></h4>

#### Parameters:
| Name       | Type                | Default                               | Description                                                         |
| ---------- | ------------------- | ------------------------------------- | ------------------------------------------------------------------- |
| strength   | <code>number</code> | <code>8</code>                        | The strength of the blur filter.                                    |
| quality    | <code>number</code> | <code>4</code>                        | The quality of the blur filter.                                     |
| resolution | <code></code>       | <code>Filter.defaultResolution</code> | The resolution of the blur filter.                                  |
| kernelSize | <code>number</code> | <code>5</code>                        | The kernelSize of the blur filter.Options&#58; 5, 7, 9, 11, 13, 15. |

</article>

## Members
<article>

### autoFit 
**Type:** <code>boolean</code>

**Default Value**: <code>true</code>

**Description:** If enabled, PixiJS will fit the filter area into boundaries for better performance. Switch it off if it does not work for specific shader.

</article>

<article>

**Default Value**: <code>PIXI.BLEND_MODES.NORMAL</code>

<article>

### blur 
**Type:** <code>number</code>

**Default Value**: <code>2</code>

**Description:** Sets the strength of both the blurX and blurY properties simultaneously

</article>


---------------------------------------

<article>

### blurX 
**Type:** <code>number</code>

**Default Value**: <code>2</code>

**Description:** Sets the strength of the blurX property

</article>


---------------------------------------

<article>

### blurY 
**Type:** <code>number</code>

**Default Value**: <code>2</code>

**Description:** Sets the strength of the blurY property

</article>


---------------------------------------

<article>

### enabled 
**Type:** <code>boolean</code>

**Description:** If enabled is true the filter is applied, if false it will not.

</article>


---------------------------------------

<article>

### legacy 
**Type:** <code>boolean</code>

**Description:** Legacy filters use position and uvs from attributes (set by filter system)

</article>


---------------------------------------

<article>

### multisample 
**Type:** <code>MSAA_QUALITY</code>

**Description:** The samples override of the filter instance.

</article>


---------------------------------------

<article>

### padding 
**Type:** <code>number</code>

**Description:** The padding of the filter. Some filters require extra space to breath such as a blur. Increasing this will add extra width and height to the bounds of the object that the filter is applied to.

</article>


---------------------------------------

<article>

### program 
**Type:** <code>Program</code>

**Description:** Program that the shader uses.

</article>


---------------------------------------

<article>

### quality 
**Type:** <code>number</code>

**Default Value**: <code>1</code>

**Description:** Sets the number of passes for blur. More passes means higher quality bluring.

</article>


---------------------------------------

<article>

### repeatEdgePixels 
**Type:** <code>boolean</code>

**Default Value**: <code>false</code>

**Description:** If set to true the edge of the target will be clamped

</article>


---------------------------------------

<article>

### resolution 
**Type:** <code>number</code>

**Description:** The resolution of the filter. Setting this to be lower will lower the quality but increase the performance of the filter.

</article>


---------------------------------------

<article>

### state 
**Type:** <code>State</code>

</article>

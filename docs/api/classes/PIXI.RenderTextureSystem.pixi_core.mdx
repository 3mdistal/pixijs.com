[@pixi/core](../modules/pixi_core).RenderTextureSystem

System plugin to the renderer to manage render textures.
Should be added after FramebufferSystem

### Frames

The `RenderTextureSystem` holds a sourceFrame â†’ destinationFrame projection. The following table explains the different coordinate spaces used&#58;

&#124; Frame            &#124; Description                                                                                   &#124; Coordinate System                                                                                                                                                                                &#124;
&#124; ---------------- &#124; --------------------------------------------------------------------------------------------- &#124; ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ &#124;
&#124; sourceFrame      &#124; The rectangle inside of which display-objects are being rendered                              &#124; **World Space**&#58; The origin on the top-left                                                                                                                                                      &#124;
&#124; destinationFrame &#124; The rectangle in the render-target (canvas or texture) into which contents should be rendered &#124; If rendering to the canvas, this is in screen space and the origin is on the top-left. If rendering to a render-texture, this is in its base-texture's space with the origin on the bottom-left. &#124;
&#124; viewportFrame    &#124; The framebuffer viewport corresponding to the destination-frame                               &#124; **Window Coordinates**&#58; The origin is always on the bottom-left.                                                                                                                                 &#124;

### viewportFrame: <span><code>Rectangle</code></span>
The viewport frame for the render-target's viewport binding. This is equal to the destination-frame for render-textures, while it is y-flipped when rendering to the screen (i.e. its origin is always on the bottom-left).

## Methods
<article>

### bind: <span><code>(renderTexture&#58; RenderTexture, sourceFrame&#58; Rectangle, destinationFrame&#58; Rectangle) -&#62; void</code></span>
Bind the current render texture.

#### Parameters:
| Name             | Type                       | Description                                                            |
| ---------------- | -------------------------- | ---------------------------------------------------------------------- |
| renderTexture    | <code>RenderTexture</code> | RenderTexture to bind, by default its `null` \- the screen.            |
| sourceFrame      | <code>Rectangle</code>     | Part of world that is mapped to the renderTexture.                     |
| destinationFrame | <code>Rectangle</code>     | Part of renderTexture, by default it has the same size as sourceFrame. |
</article>


---------------------------------------

<article>

### clear: <span><code>(clearColor&#58; ColorSource, mask&#58; BUFFER_BITS) -&#62; void</code></span>
Erases the render texture and fills the drawing area with a colour.

#### Parameters:
| Name       | Type                     | Description                                                                                      |
| ---------- | ------------------------ | ------------------------------------------------------------------------------------------------ |
| clearColor | <code>ColorSource</code> | The color as rgba, default to use the renderer backgroundColor                                   |
| mask       | <code>BUFFER_BITS</code> | Bitwise OR of masks that indicate the buffers to be cleared, by default COLOR and DEPTH buffers. |
</article>

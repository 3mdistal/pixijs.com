Format when registering an extension. Generally, the extension should have these values as `extension` static property, but you can override name or type by providing an object.



        

## Summary

<details><summary>Properties</summary>
<p>

| Name     | Type                                              | Description                                                                   |
| -------- | ------------------------------------------------- | ----------------------------------------------------------------------------- |
| name     | <code>string</code>                               | Optional. Some plugins provide an API name/property, such as Renderer plugins |
| priority | <code>number</code>                               | Optional, used for sorting the plugins in a particular order                  |
| ref      | <code>any</code>                                  | Reference to the plugin object/class                                          |
| type     | <code>ExtensionType &#124; ExtensionType[]</code> | The extension type, can be multiple types                                     |

</p>
</details>
